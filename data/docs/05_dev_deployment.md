# 개발 배포 가이드

## 배포 환경
- 개발(dev): 자유 배포, 모든 개발자 권한
- 스테이징(stg): 팀장 승인 필요
- 운영(prod): 코드리뷰 2명 + 팀장 승인 + 변경 관리 티켓

## 배포 도구
- CI/CD: GitHub Actions
- 컨테이너 레지스트리: 사내 Harbor
- 오케스트레이션: Kubernetes (사내 클러스터)
- 모니터링: Datadog, Grafana

## 배포 시간 제한
- 운영 배포는 평일 10:00~16:00에만 허용
- 금요일 오후 및 공휴일 전날 배포 금지
- 긴급 핫픽스는 CTO 승인 후 가능

## 롤백 정책
- 배포 후 30분 이내 이상 징후 시 자동 롤백
- 수동 롤백은 배포 담당자가 즉시 수행
- 롤백 후 사고 보고서 24시간 내 작성

## 시크릿 관리
- 모든 시크릿은 HashiCorp Vault 사용
- 코드/설정 파일에 하드코딩 절대 금지
- 발견 즉시 보안팀 신고 → 키 폐기 → 재발급
